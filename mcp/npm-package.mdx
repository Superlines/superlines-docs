The `@superlines/mcp-server` npm package implements the [Model Context Protocol](https://modelcontextprotocol.io) for Superlines AI visibility analytics. Connect Claude Desktop, Cursor, or any MCP-compatible client to your Superlines data.

## Features

- **Brand Analytics** â€” Visibility metrics, citations, share of voice
- **Competitor Analysis** â€” Benchmark against competitors with sentiment analysis
- **Citation Tracking** â€” See which URLs are cited in AI responses
- **Trend Analysis** â€” Weekly and monthly performance trends
- **Webpage Auditing** â€” LLM-friendliness analysis for any URL
- **Schema Optimization** â€” Generate optimized Schema.org markup
- **Strategic Action Plans** â€” Priority-ranked recommendations with improvement potential scores

## Prerequisites

- **Node.js 18+** installed ([download](https://nodejs.org/))
- **Superlines account** with a paid plan
- **API Key** from your [Superlines dashboard](https://analytics.superlines.io/settings/api-keys)

## Installation

### Option A: Use via npx (recommended)

No installation needed â€” npx runs the latest version automatically:

```json
{
  "mcpServers": {
    "superlines": {
      "command": "npx",
      "args": ["-y", "@superlines/mcp-server"],
      "env": {
        "SUPERLINES_API_KEY": "sl_live_YOUR_API_KEY"
      }
    }
  }
}
```

### Option B: Global installation

```bash
npm install -g @superlines/mcp-server
```

Then use this configuration:

```json
{
  "mcpServers": {
    "superlines": {
      "command": "superlines-mcp",
      "env": {
        "SUPERLINES_API_KEY": "sl_live_YOUR_API_KEY"
      }
    }
  }
}
```

### Option C: Local proxy script

If you prefer to download and run the proxy script locally:

<Tabs>
  <Tab title="macOS / Linux">
    ```bash
    mkdir -p ~/superlines-mcp
    curl -o ~/superlines-mcp/mcp-proxy.js \
      https://raw.githubusercontent.com/Superlines/mcp-server/main/mcp-proxy.js
    chmod +x ~/superlines-mcp/mcp-proxy.js
    ```

    Config:
    ```json
    {
      "mcpServers": {
        "superlines": {
          "command": "node",
          "args": ["/Users/YOUR_USERNAME/superlines-mcp/mcp-proxy.js"],
          "env": {
            "SUPERLINES_API_KEY": "sl_live_YOUR_API_KEY"
          }
        }
      }
    }
    ```
  </Tab>
  <Tab title="Windows">
    ```powershell
    mkdir $env:USERPROFILE\superlines-mcp
    Invoke-WebRequest -Uri "https://raw.githubusercontent.com/Superlines/mcp-server/main/mcp-proxy.js" `
      -OutFile "$env:USERPROFILE\superlines-mcp\mcp-proxy.js"
    ```

    Config:
    ```json
    {
      "mcpServers": {
        "superlines": {
          "command": "node",
          "args": ["C:\\Users\\YOUR_USERNAME\\superlines-mcp\\mcp-proxy.js"],
          "env": {
            "SUPERLINES_API_KEY": "sl_live_YOUR_API_KEY"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

<Warning>
Replace `YOUR_USERNAME` with your actual username. Find it by running `echo $USER` (macOS) or `echo %USERNAME%` (Windows). Always use full absolute paths â€” not `~`.
</Warning>

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `SUPERLINES_API_KEY` | Yes | Your Superlines API key (starts with `sl_live_`) |
| `MCP_SERVER_URL` | No | Override the default server URL |
| `DEBUG` | No | Set to `true` for debug logging |

## Troubleshooting

<AccordionGroup>
  <Accordion title="SUPERLINES_API_KEY environment variable not set">
    Make sure your config includes the `env` section with your API key. The key must start with `sl_live_`.
  </Accordion>
  <Accordion title="401 Unauthorized">
    Your API key is invalid or expired. Generate a new key at [Organization Settings â†’ API Keys](https://analytics.superlines.io/settings/api-keys).
  </Accordion>
  <Accordion title="403 Forbidden">
    API/MCP access requires a paid plan. Contact [support@superlines.io](mailto:support@superlines.io) to upgrade.
  </Accordion>
  <Accordion title="Claude says 'I don't have access to any tools'">
    1. **Restart Claude Desktop completely** (Cmd+Q on Mac, not just close window)
    2. **Check the script path** â€” use the full absolute path, not `~`
    3. **Validate your config** is valid JSON:
       ```bash
       cat ~/Library/Application\ Support/Claude/claude_desktop_config.json | python3 -m json.tool
       ```
    4. **Test the proxy manually:**
       ```bash
       export SUPERLINES_API_KEY="sl_live_your_key"
       echo '{"jsonrpc":"2.0","method":"tools/list","id":1}' | npx -y @superlines/mcp-server
       ```
       You should see a JSON response listing available tools.
  </Accordion>
</AccordionGroup>

## Security

<Warning>
- Never share your API key publicly
- Don't commit `claude_desktop_config.json` to version control
- Rotate keys regularly in Organization Settings
- Revoke keys immediately if compromised

Secure your config file:
```bash
chmod 600 ~/Library/Application\ Support/Claude/claude_desktop_config.json
```
</Warning>
